Transparenter HTTP-Proxy Empfohlen

Auf dem Router/Gateway einrichten — alle HTTP-Downloads aller LAN-Geräte werden automatisch gecacht, ohne dass auf den Clients etwas konfiguriert werden muss. Gilt auch für Geräte ohne Proxy-Unterstützung.

Standard-Modus ({{ standard_ip }}): HTTP wird gecacht, HTTPS wird transparent durchgeleitet. Kein CA-Zertifikat auf den Clients nötig.

iptables (auf dem Router/Gateway als root ausführen)

# Transparenter HTTP-Cache — alle LAN-Anfragen auf Port 80 abfangen
iptables -t nat -A PREROUTING -i LAN_INTERFACE -p tcp --dport 80 \
  ! -d {{ standard_ip }} -j DNAT --to-destination {{ standard_ip }}:80

# Beim Neustart persistent machen (Debian/Ubuntu)
apt install iptables-persistent
netfilter-persistent save

LAN_INTERFACE durch das interne Netz-Interface ersetzen (z. B. eth1, br0).

nftables (moderner Ersatz für iptables)

nft add table ip lancache
nft add chain ip lancache prerouting \
  '{ type nat hook prerouting priority -100; }'
nft add rule ip lancache prerouting \
  iifname "LAN_INTERFACE" tcp dport 80 \
  ip daddr != {{ standard_ip }} \
  dnat to {{ standard_ip }}:80
Rückgängig machen (iptables) 
iptables -t nat -D PREROUTING -i LAN_INTERFACE -p tcp --dport 80 \
  ! -d {{ standard_ip }} -j DNAT --to-destination {{ standard_ip }}:80

Client-seitige Proxy-Einstellung Alternativ

Ohne Router-Zugang: Proxy direkt im Paketmanager oder als Systemvariable konfigurieren. Muss auf jedem Gerät einzeln eingerichtet werden.

apt (Debian · Ubuntu · Pop!_OS · Mint)

echo 'Acquire::http::Proxy "http://{{ standard_ip }}/";' \
  | sudo tee /etc/apt/apt.conf.d/01lancache

pacman / yay (Arch · CachyOS · Garuda · Manjaro)

echo 'http_proxy=http://{{ standard_ip }}' | sudo tee -a /etc/environment
echo 'https_proxy=http://{{ standard_ip }}' | sudo tee -a /etc/environment

pacman hat keinen eingebauten Proxy — die Env-Variable wird vom Downloader (curl/wget) ausgewertet.

dnf (Fedora · Nobara · AlmaLinux · Rocky)

echo 'proxy=http://{{ standard_ip }}' | sudo tee -a /etc/dnf/dnf.conf

pip (Python-Pakete)

pip config set global.proxy http://{{ standard_ip }}

SSL-Modus ({{ ssl_ip }}) — auch HTTPS cachen

Clients müssen das CA-Zertifikat einmalig installieren. Danach werden auch HTTPS-Downloads gecacht. DNS-Server auf {{ ssl_ip }} setzen (statt {{ standard_ip }}).

CA-Zertifikat installieren

# Linux (systemweit)
sudo cp ca.crt /usr/local/share/ca-certificates/lancache-ca.crt
sudo update-ca-certificates

# Arch / CachyOS / Garuda (trust-store)
sudo trust anchor --store ca.crt

Vollständige Anleitung für alle Betriebssysteme: docs/install-ca-cert.md
