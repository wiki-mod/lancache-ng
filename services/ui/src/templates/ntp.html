LanCache-NG-NTP

Ein eigener NTP-Server, der seine Zeit gegen mehrere öffentliche NTP-Server synchronisiert und sie im LAN auf UDP/123 bereitstellt.

Status

Startet/stoppt den `ntp`-Container. Erfordert, dass das `ntp`-Compose-Profil beim letzten Setup aktiviert wurde (siehe setup.sh) -- ist der Container nie erstellt worden, schlägt das Aktivieren mit einer klaren Fehlermeldung fehl.

{% if ntp_enabled %}aktiviert{% else %}deaktiviert{% endif %}

Leerzeichen- oder kommagetrennt. Hostnamen oder IPv4/IPv6-Adressen. Mindestens ein Eintrag ist erforderlich -- dieser Dienst läuft nie als eigenständige Zeitquelle.

Nur wirksam, wenn LanCache-NG-NTP aktiviert UND der DHCP-Server (Kea) aktiv ist. Solange dieser Schalter an ist, überschreibt er das ntp-servers-Feld JEDES Kea-Subnetzes mit der obigen LAN-Adresse -- ein manuell pro Subnetz gesetzter Wert wird dabei nicht erhalten. Beim Ausschalten kehren alle Subnetze zum projektweiten Standardwert zurück (siehe DHCP-Seite, Feld "NTP servers").

{% if not dhcp_has_kea %}

DHCP läuft derzeit nicht im Kea-Modus -- dieser Schalter greift erst, sobald Kea aktiv ist.
