Neuen Secondary registrieren

Befehl auf dem Secondary-Host ausführen — ersetzt myname durch einen eindeutigen Namen. Der Installer bindet standardmäßig an die erkannte LAN-IP des Hosts; wenn Port 53 dort bereits belegt ist, zeigt er den belegenden Socket an und fragt nach einer alternativen --listen-ip.

bash <(curl -fsSL https://raw.githubusercontent.com/wiki-mod/lancache-ng/master/setup.sh) secondary \ --primary {{ primary_url }} \ --token {{ registration_token }} \ --name myname \ --proxy-ip {{ primary_url | replace(from="http://", to="") | replace(from=":8080", to="") }}

Der Befehl registriert den Secondary, generiert die benötigten Werte und startet den Container automatisch.

Hinweis für lokale Tests auf demselben Host: Wenn bereits ein Dienst Port 53 belegt, musst du eine freie Bind-IP per --listen-ip angeben. Der Installer zeigt dir den Konflikt dann direkt an, statt erst beim Docker-Start zu scheitern.

Zeitangaben werden als lokale Datum-/Uhrzeitangaben dargestellt, nicht als Unix-Zeitstempel.
