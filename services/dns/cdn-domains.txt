# lancache-ng (https://github.com/wiki-mod/lancache-ng)
# CDN domain list for DNS spoofing. One domain per line, three explicit match modes (#1072):
#   domain.com      -> matches ONLY domain.com (no subdomains)
#   .domain.com     -> matches ONLY *.domain.com (every subdomain, not the bare root)
#   sub.domain.com  -> matches ONLY that exact host (no further subdomains beneath it)
# A bare entry never implicitly also wildcards what's underneath it, regardless of how
# many labels it has. Lines starting with # are ignored.

# LanCache DNS — CDN domain list
# Prefix a line with "!" to disable it without deleting it (e.g. "!steamcontent.com"
# or "!.steamcontent.com") -- both DNS RPZ generation (services/dns/entrypoint.sh)
# and proxy cert/nginx-map generation (services/proxy/entrypoint.sh) skip a
# disabled entry entirely. Remove the "!" to re-enable it. The Admin UI's
# per-domain toggle (services/ui/src/routes/domains.rs) manages this marker
# for the pre-shipped entries below; do not hand-edit the "====" marker line
# further down this file, it tells the Admin UI where operator-added entries
# start.
