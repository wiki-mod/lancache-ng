# Code of Conduct

## Our standard

The lancache-ng project should be a practical and respectful place to improve
network caching software together.

Examples of welcome behavior:

- using clear and respectful language
- accepting that people have different experience levels
- giving technical feedback with enough context to act on it
- focusing criticism on code, behavior, documentation or risk
- helping keep discussions useful for maintainers and operators

Examples of unacceptable behavior:

- harassment, insults, threats or personal attacks
- sexualized language or imagery
- public or private intimidation
- publishing another person's private information
- repeated off-topic disruption after being asked to stop
- knowingly sharing secrets, credentials, private certificates or private
  operational data from another installation

## Scope

This code of conduct applies in project spaces, including issues, pull
requests, discussions, reviews, documentation and other collaboration channels
used for lancache-ng.

It also applies when someone represents the project in public spaces.

## Reporting

If you see or experience unacceptable behavior, report it to the project
maintainers.

When reporting, include the relevant links, screenshots or context if you can
do so safely. Do not publish private information in a public issue.

## Enforcement

Maintainers may take action when behavior harms the project or its community.
Possible actions include:

- asking for clarification or a change in tone
- editing or removing comments
- closing issues or pull requests
- temporary or permanent restriction from project spaces

Maintainers should apply enforcement consistently and proportionally. Serious
or repeated harmful behavior can lead to stronger action without repeated
warnings.

## Project-specific safety note

lancache-ng is used on real networks. Do not pressure contributors or operators
to publish private IP layouts, credentials, certificates, logs with secrets or
other sensitive local configuration. When sensitive information is needed for
debugging, prefer redacted examples or a private security report.
